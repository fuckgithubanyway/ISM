# DDS Manifest: Structure & Topology

**Версия:** 0.5


## Корневая Директория (`[project_root]/dds/`)
Все артефакты методологии должны располагаться в выделенной директории `[project_root]/dds/` в корне проекта.


## Принцип Зеркалирования (The Mirror Principle)
Структура директорий внутри `[project_root]/dds/` должна быть изоморфна структуре исходного кода проекта.

*   **Артефакт:** `[project_root]/src/auth/login.ts`
*   **Спецификация:** `[project_root]/dds/src/auth/login.md`

Это обеспечивает алгоритмическую детерминированность поиска требований: Агент всегда знает, где искать Спецификацию для любого файла.


## Типология Спецификаций

### Манифест-файлы (`-manifest-[name].md`)
*   **Расположение:** Только корень `[project_root]/dds/`.
*   **Назначение:** Определяют фундаментальные правила методологии. Файлы этой категории формируют конституционный слой проекта. Их изменение меняет фундаментальные правила работы над проектом.

### Мета-спецификации (`-meta-[name].md`)
*   **Расположение:** Любая директория внутри `[project_root]/dds/`.
*   **Назначение:** Определяют контекст, правила и ограничения (Tech Stack, Code Style, Testing Rules, etc).
*   **Область действия:** Распространяются на директорию размещения и все вложенные поддиректории (рекурсивно). Мета-спецификации поддиректорий имеют приоритет над родительскими для всех Артефактов поддиректории (и для всех поддиректорий).

### Функциональные Спецификации (`-spec-[name].md`)
*   **Расположение:** Внутри дерева `[project_root]/dds/`, зеркально отражая структуру исходного кода.
*   **Назначение:** Описывают бизнес-логику, требования и контракты конкретного артефакта или модуля.

### Спецификации Реализации (`-spec-[name].[ext].md`)
*   **Расположение:** Внутри дерева `[project_root]/dds/`, зеркально отражая структуру исходного кода.
*   **Назначение:** Описывают технические детали реализации для конкретного языка или технологии.
*   *Пример:* `user.py.md` содержит специфику Python реализации конкретного Артефакта, в то время как `user.md` содержит общую логику.


## Спецификация Модулей (Директорий)
Для описания назначения целой директории (например, `[project_root]/src/auth/`) используется файл с именем директории и префиксом `-meta-`, расположенный внутри соответствующей директории DDS.

*   **Путь:** `[project_root]/dds/src/auth/-meta-auth.md`
*   **Назначение:** Описывает архитектурную роль модуля, его публичный API и ответственность.
*   **Делегирование:** Если в директории присутствует `-meta-[dirname].md`, но отсутствуют спецификации для вложенных файлов, Агент обязан самостоятельно реализовать содержимое модуля, руководствуясь общими требованиями к модулю (принцип "Черного ящика").
