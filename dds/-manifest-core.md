# DDS Manifest: Core Axioms

**Версия:** 0.5


## Определение
Development by Declarative Specification (DDS) — Разработка на основе Декларативных Спецификаций — методология разработки, при которой Артефакты проекта генерируются исполняющим Агентом из декларативных Спецификаций на естественном языке.


## Аксиомы

### Примат Спецификации (Primacy of Specification)
Спецификация имеет абсолютный приоритет над кодом.
*   Если поведение кода противоречит Спецификации, это **Ошибка кода**.
*   Если требуется изменить поведение системы, сначала обновляется Спецификация, и только затем — Код.
*   **Запрещено** изменять логику в коде без предварительного отражения изменений в Спецификации (Spec Drift).

### Итеративность (Iterative Refinement)
DDS поддерживает жизненный цикл ПО. Спецификации не являются статичными; они эволюционируют вместе с проектом. Разрешается и поощряется уточнение требований от абстрактных к конкретным по мере разработки.

### Частичное Покрытие (Partial Coverage)
DDS допускает существование Артефактов, не имеющих прямой Функциональной Спецификации.
*   Такие Артефакты называются **Неспецифицированными (Unspecified Artifacts)**.
*   Присутствие Неспецифицированных Артефактов не нарушает целостность проекта, но ограничивает возможности Агента по гарантии корректности изменений.
