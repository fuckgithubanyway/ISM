# Projective Development Standard (PDS)

**Версия:** 0.1
**Статус:** Draft / RFC

**PDS** — это методология разработки программного обеспечения, постулирующая, что исходный код является транзиентной (временной) проекцией, синтезируемой из структурированных намерений и спецификаций.

## Навигация по стандарту

Стандарт формально описан внутри директории `pds/` в виде набора Манифестов. Это "Конституция" методологии.

### [📂 Перетий к файлам стандарта (pds/)](./pds/)

*   [**Core Axioms**](./pds/-manifest-core.md) (`-manifest-core.md`)
    *   Философия, определения (Зона Определения, Зона Проекции) и аксиоматика.
*   [**Topology & Artifacts**](./pds/-manifest-topology.md) (`-manifest-topology.md`)
    *   Файловая структура, типы спецификаций, принцип изоморфизма.
*   [**Workflow & Protocols**](./pds/-manifest-workflow.md) (`-manifest-workflow.md`)
    *   Режимы работы (Синтез, Реконсиляция), протокол ADR и разрешение конфликтов.
