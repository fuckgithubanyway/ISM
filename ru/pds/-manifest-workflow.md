# PDS Manifest: Workflows & Protocols

**Версия:** 0.2
**Язык:** Русский


## Каскад Приоритетов

При возникновении противоречий (Коллизий) истиной считается источник с высшим приоритетом:

1.  **Манифест** (Методологический фундамент).
2.  **Мета-Спецификация** (Локальный контекст с учетом вложенности).
3.  **Спецификация Реализации** (Физический скелет).
4.  **Функциональная Спецификация** (Бизнес-логика).
5.  **Намерение пользователя** (Текущий запрос).

*@ADR: Высший приоритет Манифеста гарантирует, что PDS-Агент не сможет нарушить методологические принципы PDS (например, изменить логику в обход спецификации) даже при прямом запросе пользователя.


## Режимы Работы

### Синтез (Synthesis): `PDS` -> `Code`
1.  PDS-Агент читает каскад (Manifest + Meta + Impl Spec + Func Spec).
2.  PDS-Агент генерирует код в Зоне Проекции.
3.  **Правило Слияния:** Блока текста (кода) из *Спецификации Реализации* копируется дословно (Immutable). Разрешение синтаксических конфликтов при слиянии *Функциональной Спецификации* и *Спецификации Реализации* является зоной ответственности PDS-Агента. При неспособности PDS-Агента безопасно интегрировать логику *Спецификации Реализации*, PDS-Агент прерывает Синтез текущего узла и запрашивает ручную корректировку Спецификации реализации у Архитектора.
*   *Область действия:* Применяется исключительно к файлам, имеющим соответствующие записи в `[root]/pds/`.
*   *Атомарность:* PDS-Агент должен стремиться к минимально необходимому изменению кода Проекции для соответствия Спецификации.

### Реконсиляция (Reconciliation): `Code` -> `PDS`
Используется при "ручном" изменении кода (хотфикс, отладка).
1.  PDS-Агент детектирует дрейф (Drift) между Проекцией и Определением.
2.  PDS-Агент формирует предложение по обновлению Спецификаций.
3.  Разработчик обязан зафиксировать обоснование согласно протоколу ADR (см. `-manifest-adr.md`).
*   *Статус:* Критическая операция с высоким риском семантической деградации.
*   *Протокол:* После выполненной PDS-Агентом Реконсиляции Архитектор обязан валидировать обновленные Функциональные Спецификации, восстановив, в случае необходимости, утраченный бизнес-контекст.
*@ADR: Определение Реконсиляции как рискованной операции сознательно усложняет процесс "обратной правки", стимулируя разработчика работать в парадигме Specification-First.

### Верификация (Verification)
Автоматическая проверка соответствия Проекции Определению. Любое отклонение, не зафиксированное в Спецификации (отсутствующее в Impl Spec и противоречащее Func Spec), считается ошибкой. При конфликте PDS-Агент генерирует отчет об ошибке (Collision Report) с указанием противоречащих узлов. Разрешение конфликта — эксклюзивная зона ответственности Архитектора путем корректировки Зоны Определения.
