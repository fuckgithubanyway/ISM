# PDS Manifest: Topology & Artifacts

**Версия:** 0.3
**Язык:** Русский


## Структурный Изоморфизм (Structural Isomorphism)

Структура Зоны Определения зеркально повторяет структуру всего проекта относительно корня. Путь `[root]/pds/foo/bar` определяет путь к артефакту `[root]/foo/bar` (или группе связанных артефактов).

*@ADR: Зеркалирование всего корня `[root]/`, а не только папки исходного кода, выбрано для обеспечения унифицированного управления всем проектом через единый семантический протокол.

*@ADR: Дублирование файловой структуры (Изоморфизм) является осознанным компромиссом. Издержки на синхронизацию путей окупаются возможностью независимой трансляции PDS-проекта в различные технологические стеки без изменения Зоны Определения.


## Классификация Спецификаций

В Зоне Определения (`[root]/pds/`) допускаются файлы следующих типа:

### Манифест (`-manifest-[*name].md`)
*   **Локация:** Только корень `[root]/pds/`.
*   **Роль:** Определение методологии и глобальных законов проекта.
*   **Примечание:** Может существовать в анонимной форме `-manifest-.md`.

### Мета-Спецификация (`-meta-[*name].md`)
*   **Локация:** Любая директория.
*   **Роль:** Контекстные ограничения (Constraints) для текущей директории и поддиректорий.
*   **Пример:** Стек технологий, Code Style, паттерны проектирования, запрещенные функции.
*   **Иерархическое Наследование:** Правила Мета-спецификаций распространяются на текущую директорию и все вложенные поддиректории. В случае конфликта приоритет имеет спецификация с максимальной вложенностью (ближайшая к целевому файлу).
*   **Примечание:** Следует избегать лишней вложенности перегружающихся Мета-Спецификаций, для оптимизации контекста.
*   **Примечание:** Может существовать в анонимной форме `-meta-.md`.
*@ADR: Каскадное наследование мета-спецификаций позволяет определять локальные технологические исключения (например, особый стек для микросервиса) без усложнения глобальных правил проекта.

### Функциональная Спецификация (`-func-[name].md`)
*   **Локация:** Изоморфно целевому модулю.
*   **Роль:** Описание логики ("Смысл").
*   **Отношение:** 1 к N (одна спека может описывать логический модуль из нескольких файлов).

### Спецификация Реализации (`-impl-[name].[ext]`)
*   **Локация:** Изоморфно целевому файлу.
*   **Формат:** Исходный текст (код) в целевом формате (на целевом языке).
*   **Роль:** Синтаксическая инъекция (по аналогии с ассемблерными вставками в Си).
*   **Содержание:** Блоки текста или программного кода (структуры, функции, отдельные выражения, демонстрирующие конкретные решения, алгоритмы и т.п.), сопровождающиеся пояснительными комментариями. Любой фрагмент текста (или программного кода) в Спецификации Реализации, имеет статус `Immutable`. PDS-Агент обязан перенести этот фрагмент в Проекцию дословно. Модификация этих участков PDS-Агентом запрещена, кроме случаев явной команды в режиме Реконсиляции.

### Промт-файлы (`-prompt-[name].md`)
*   **Локация:** Любая директория.
*   **Роль:** Используются для конфигурирования среды или IDE.
*   **Примечание:** Промт-файлы не являются спецификациями.
