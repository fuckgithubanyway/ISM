# Development by Declarative Specification (DDS)

**DDS** — методология разработки ПО, инвертирующая классический процесс производства кода. В парадигме DDS исходный код теряет статус первичного источника истины и становится производным Артефактом, автоматизированно синтезируемым на основе иерархии декларативных Спецификаций.

Проект представляет собой стандарт протокола взаимодействия между Оператором (человеком) и Интерпретатором (LLM), а также референсную реализацию инструментария для управления контекстом.

## Каноническая спецификация

Исчерпывающее описание онтологии, топологии проекта, формата метаданных и протокола директив содержится в Манифесте:

> **[Спецификация протокола DDS (Манифест)](./dds/specs/__dds.spec.md)**

*Любая реализация, претендующая на совместимость с DDS, обязана строго следовать положениям данного документа.*

## Структура репозитория

*   **`dds/specs/`** — Хранилище эталонных спецификаций.
    *   `__dds.spec.md` — Манифест (Root Specification).
*   **`dds/tools/`** — Референсная реализация CLI-инструментов (POSIX shell) для формирования Снимков (`snapshot`), валидации (`verify`) и управления проекциями (`list`).
